## 以下是我的理解
Navier–Stokes方程描述整个速度场随时间变化的演化规则，是一个微分物理公式。
如果我成功建立了从(x,y,z,t)计算(u,v,w)的神经网络，也建立了从(x,y,z,t)计算pressure，我就应该能够计算出Navier–Stokes方程里面所有的物理量（比如偏导数的计算），并将这些物理量带入Navier–Stokes方程使得等号成立。除了Navier–Stokes方程，神经网络也要满足边界条件：比如入口的风速、管壁的风速、散热器边上的风速。
这个公式里面也包含其他基础的物理信息比如密度和散度，需要被代入公式。
本质上PINNS就是通过将神经网络所构成的映射函数(x,y,z,t)->(u,v,w)，(x,y,z,t)->pressure，将其带入微分方程，目标是使得等号成立，等式的左边-右边，就是用来训练网络的loss。

通过微分方程计算出来loss是模型最主要的约束条件（Constraint），也就是interior_flow，还有其他的边界条件比如：
- inlet: 入口处规定流速剖面（抛物线）、无纵向分量、温度为环境温度。这是流体从左侧进入通道时的已知状态
- outlet: 出口处压力设为 0（参考压力基准）。流体自由流出，不对速度施加约束
- hs_wall: 散热片表面：无滑移（流体贴壁静止）+ 固定高温。代表热源，是热量输入的起点
- channel_wall：通道上下壁面：无滑移 + 绝热（法向温度梯度为 0，热量不穿过通道壁散走）
还有补充约束：
- integral_continuity: 任意截面的总质量流量守恒（不可压缩流体）。

用于训练的数据，来源于在domain里面进行点的采集，分别在inlet、outlet、hs_wall、channel_wall、interior_flow等位置进行数据采集，本质上就是获得(x,y,z)及相关的基础信息。
最终将所有采集点的loss计算出来加起来用于训练模型。

heat_sink.py是基于这个原理的一个系统简化：
- 求最终稳态的速度场，对于神经网络的变动时把输入的t去除，对于代入公式的改动是取消速度对时间的导数，因为速度不再变化。
- 从三维简化为二维，对于神经网络的变动时把输入的z去除，对于代入公式的改动是把与z相关的导数去除。

我们还可以考虑将模型（神经网络）复杂化，比如我想把入口的风速作为一个参数/变量输入给模型，这种情况下的改动就是：
1. 对于模型的变动是输入参数加入inlet_vel：(x,y,z,t,inlet_vel)->(u,v,w)
2. 在采集点的时候采集的信息要增加一个维度：(x,y,z,inlet_vel)，也就意味着在训练的时候增加了一个维度
经验上，增加一个参数维度，训练步数通常需要增加 2~5 倍，具体取决于参数范围宽窄和精度要求。
